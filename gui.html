GUI & Use

Capturing frames

First we start Brofiler and we can see the following interface.


When Brofiler is already open, we have to execute our application in which we have implemented it previously.

Now we can capture the frames by clicking on the red button on the upper left corner.

If everything has gone well in the timeline will appear a counter with the frames that the program is capturing.



When we consider that we have enough frames click on the red square . Brofiler will stop capturing frames and show us the information of the data it has collected.

Viewing the data

After stopping frame capture, Brofiler will display a bar in the timeline for each frame it has captured with the runtime. Clicking on any of the bars will load the data of the threads and the functions that have been executed in that frame into a new new tab at the bottom. We can use the mouse wheel to zoom in or out in the threads window.




On the right side we can see a table that shows all the functions executed in that frame with the following information:

  • S: checkbox to mark the function.
  • Function: the name of the function.
  • Self% the execution time percentage that has used the function in that frame.
  • Self(ms): the time it has taken this function to run without counting other functions inside it.
  • Total(ms): the sum of milliseconds that has consumed the function among all its executions.
  • Max(ms): the highest execution time that has been recorded for all executions in that frame.
  • Count: the number of times the function has been executed in that frame.
  • Path: the path from which the function is executed.


Above the table there is a searcher to filter the rows by the name of the function.

Double click on any row in the table will open Source View window.

The hierarchy of function calls is shown on the left side with the runtime. Above the hierarchy we can find the following buttons:

  • Hides all nodes with small time: it will stop showing the functions that have a run time inferior to the one introduced in the textbox.
  • Focus on callstacks: it shows only the hierarchy from the function that we select in the right section.
  • Show events in percent: change the time in milliseconds by the percentage.
  • Show source path: displays the path next to the function name.

Sampling & Hooking

If we activate a checkbox of the table, in the next data collection we will sample the whole code within the scope of the selected function.

Once we have the sample frame we can activate another checkbox of the table of that frame to perform hooking, this will inject counters that will show even more information in the next collection of data.

Click on this icon to remove all the sampling scopes and on this to remove the counters injected by the hooking.

Save, Load & Clear

We can save the current session and open it later to continue working or to be able to see the difference between before and after doing an optimization. To save it we will have to click on the save button , indicate where to save it and the name of the file.

To load the .prof file that Brofiler has generated, we will have to drag it over the program and it will open automatically.

To clear the session and set Brofiler as its initial state, we will click on the trash icon .
